2 Learning Single-Cell RNA Sequencing Analysis with Seurat

2.1 Background & Motivation

Single-cell RNA sequencing (scRNA-seq) has fundamentally changed how we study gene expression. Rather than averaging signals across millions of cells in a bulk sample, scRNA-seq allows us to profile gene expression in each individual cell — revealing cell-type composition, rare populations, developmental trajectories, and disease-specific transcriptional states.

This technology is increasingly central to research in oncology (tumor heterogeneity), immunology, neuroscience, and rare disease genomics — all areas that overlap with my future ambitions in bioinformatics. In the pharmaceutical industry specifically, scRNA-seq is used for drug mechanism studies, biomarker discovery, and understanding how different cell populations respond to treatment.

Given that my Projecticum project deals with ALS genomics, where understanding cell-type-specific gene expression in motor neurons is highly relevant, learning scRNA-seq analysis is a natural and motivated next step.

2.1.1 Connection to the future plan

As described in the previous chapter, I aim to work in bioinformatics or genomics research. scRNA-seq competency is listed as a desired skill in virtually every junior bioinformatics role in the field today. This assignment gives me a structured opportunity to build that skill from the ground up — starting with a tutorial dataset and ending with an independent analysis on a novel dataset.

2.3 The scRNA-seq Pipeline with Seurat

The standard Seurat pipeline takes a count matrix (genes × cells) as input — this is the output of upstream processing tools like Cell Ranger and represents how many reads of each gene were detected per cell. The steps below describe the full analysis workflow. Tutorial can be found at https://satijalab.org/seurat/articles/pbmc3k_tutorial

2.3.1 Step 1 — Loading Data

library(Seurat)
library(ggplot2)
library(dplyr)
library(patchwork)

# Load the PBMC 3k dataset (available via Seurat tutorial)
pbmc.data <- Read10X(data.dir = "~/dsfb2_workflows_portfolio/processed_data/filtered_gene_bc_matrices/hg19/")
pbmc <- CreateSeuratObject(counts = pbmc.data, project = "pbmc3k",
                           min.cells = 3, min.features = 200)

2.3.2 Step 2 — Quality Control & Filtering

Low-quality cells are identified and removed based on three metrics:

  • nFeature_RNA — number of unique genes detected per cell (too low = dead cell; too high = doublet)
  • nCount_RNA — total number of RNA molecules per cell
  • percent.mt — percentage of reads mapping to mitochondrial genes (high = dying cell)
pbmc[["percent.mt"]] <- PercentageFeatureSet(pbmc, pattern = "^MT-")

VlnPlot(pbmc, features = c("nFeature_RNA", "nCount_RNA", "percent.mt"), ncol = 3)

pbmc <- subset(pbmc, subset = nFeature_RNA > 200 & nFeature_RNA < 2500 & percent.mt < 5)

2.3.3 Step 3 — Normalization

Each cell’s gene counts are normalized to account for differences in sequencing depth, then log-transformed.

pbmc <- NormalizeData(pbmc, normalization.method = "LogNormalize", scale.factor = 10000)

2.3.4 Step 4 — Feature Selection

Highly variable genes (HVGs) — genes that show meaningful variation across cells — are selected for downstream analysis. This reduces noise and computational load.

pbmc <- FindVariableFeatures(pbmc, selection.method = "vst", nfeatures = 2000)

2.3.5 Step 5 — Scaling

Data is scaled so that each gene has a mean of 0 and variance of 1 across cells, which is required for PCA.

all.genes <- rownames(pbmc)
pbmc <- ScaleData(pbmc, features = all.genes)

2.3.6 Step 6 — PCA

Principal component analysis reduces the high-dimensional gene expression space to a manageable number of principal components (PCs).

pbmc <- RunPCA(pbmc, features = VariableFeatures(object = pbmc))
ElbowPlot(pbmc)  # used to decide how many PCs to use

2.3.7 Step 7 — Clustering

A nearest-neighbor graph is constructed in PC space, then the Louvain algorithm identifies clusters of transcriptionally similar cells.

pbmc <- FindNeighbors(pbmc, dims = 1:10)
pbmc <- FindClusters(pbmc, resolution = 0.5)

2.3.8 Step 8 — UMAP Visualization

UMAP projects the high-dimensional data into 2D for visual exploration of clusters.

pbmc <- RunUMAP(pbmc, dims = 1:10)
DimPlot(pbmc, reduction = "umap", label = TRUE)

2.3.9 Step 9 — Marker Gene Identification

Differentially expressed genes that define each cluster are identified.

pbmc.markers <- FindAllMarkers(pbmc, only.pos = TRUE, min.pct = 0.25,
                               logfc.threshold = 0.25)

2.3.10 Step 10 — Cell Type Annotation

Marker genes are matched to known cell type signatures. For example, in PBMCs:

Cluster markers Cell type
IL7R, CCR7 Naive CD4+ T cells
CD14, LYZ CD14+ Monocytes
MS4A1 B cells
CD8A CD8+ T cells
GNLY, NKG7 NK cells 
new.cluster.ids <- c("Naive CD4 T", "CD14+ Mono", "Memory CD4 T",
                     "B", "CD8 T", "FCGR3A+ Mono", "NK", "DC", "Platelet")
names(new.cluster.ids) <- levels(pbmc)
pbmc <- RenameIdents(pbmc, new.cluster.ids)
DimPlot(pbmc, reduction = "umap", label = TRUE, pt.size = 0.5)

2.4 Key Tools & Resources

  • Seurat — main R package for scRNA-seq analysis (satijalab.org/seurat)
  • ggplot2 — visualization
  • DoubletFinder — optional doublet detection
  • SingleR — automated cell type annotation using reference datasets
  • OSCA Book — Orchestrating Single-Cell Analysis (bioconductor.org/books/release/OSCA)

Good starting datasets:

  • PBMC 3k — classic beginner dataset, available directly via the Seurat tutorial
  • CZ CELLxGENE — large curated collection of scRNA-seq datasets, recommended by professors — cellxgene.cziscience.com
  • 10x Genomics public datasets — genomics.10xgenomics.com
  • GEO (Gene Expression Omnibus) — search scRNA-seq + disease of interest

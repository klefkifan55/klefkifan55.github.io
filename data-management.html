3 Data Management

Good data management is foundational to reproducible science. In this chapter, the Guerilla Analytics principles for data management are applied to two contexts: the RNA-seq formatieve opdracht from the DAUR2 module, and my own portfolio repository.

3.1 The Guerilla Analytics Principles

Guerilla Analytics is a pragmatic framework for managing analytical data in complex, real-world environments. The core rules most relevant to bioinformatics workflows are:

  1. Space is cheap, confusion is expensive — keep raw data raw and never overwrite it
  2. Use a single, versioned directory structure — everything has a defined place
  3. Separate raw data from derived data — processed files live in separate folders
  4. Every dataset should have a README — describe what the data is, where it came from, and how to use it
  5. Automate where possible — use scripts rather than manual steps to ensure reproducibility

3.2 3a. RNA-seq Data Management (DAUR2)

The RNA-seq formatieve opdracht from the DAUR2 module involves multiple types of data files generated at different stages of the pipeline: raw reads (FASTQ), aligned reads (BAM), and count files. The Guerilla Analytics structure below organizes these files in a clear, reproducible way.

# DAUR2 RNA-seq Formatieve Opdracht

## Project description
This directory contains data and analysis scripts for the RNA-seq
formatieve opdracht from the DAUR2 module (DSFB1 cursus)

## Author
Robin Jansen | robinjansentilburg@icloud.com | github.com/klefkifan55

## Data origin
Raw FASTQ files are stored in raw_data/fastq/.
BAM files (aligned reads) are stored in raw_data/bam/.
FastQC quality control reports are stored in analysis/fastqc_HTML/.

## Directory structure
├── analysis/
│   └── fastqc_HTML/       → FastQC quality control HTML reports
├── processed_data/        → cleaned/processed output files
├── raw_data/
│   ├── fastq/             → raw sequencing reads (do not edit)
│   └── bam/               → aligned reads produced by pipeline
├── references/            → reference genome and annotation files
└── scripts/               → analysis scripts (bash and R)

## How to reproduce
Run bash scripts in order, then R scripts for downstream analysis.
All paths are relative to the project root.

## How to build
Set working directory to scripts/ and run in the R console:
  bookdown::render_book('.')

The rendered site is output to scripts/_book/.
Copy the contents of _book/ to the klefkifan55.github.io repository and push to update the live site.
